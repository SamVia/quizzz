## **Major Topics: Introducing the Network Layer in Enterprise Networks**

### **1. The Role of Routers in LANs**
Routers are mandatory devices in any Local Area Network (LAN) and traditionally serve two distinct functions:
* **Border Router:** Connects the corporate infrastructure to the Internet.
* **Internal (One-Arm) Router:** Connects hosts and servers residing in different VLANs.
* **Implementation:** For practical reasonsâ€”such as the availability of different physical interfaces, required forwarding speeds, and the necessity of different routing protocolsâ€”these two functions are usually implemented using two separate devices.

**The Fundamental Design Question:**
Where do we stop L2 and start L3 in the network?  Options include:
* At the edge of the corporate network (exit gateway).
* In the core of the corporate network (backbone).
* In the distribution layer of the corporate network.
* In the access layer (e.g., putting all hosts within different `/30` networks), utilizing no L2 at all.

---

### **2. Layer 2 vs. Layer 3: Comprehensive Comparison**

The choice between staying with L2 or moving to L3 revolves around several architectural trade-offs.

#### **A. Simplicity and Cost**
* **Layer 2 (L2):** Offers transparency and simplicity. It acts as "plug and play" deployment with no configuration needed for the network or the host. It is highly performant, simple to produce, and very cheap.
* **Layer 3 (L3):** Both hosts and routers must be configured. It is more expensive and has slower processing speeds because it requires extracting layer 3 packets, updating them, and building new layer 2 frames. It also requires longest prefix matching and sophisticated routing protocols.

#### **B. Addressing and Scalability**
* **L2:** Relies on flat addressing. Each reachable host must be explicitly listed in the filtering database.
* **L3:** Uses hierarchical addressing where a single entry can forward data to multiple destinations (e.g., destinations sharing the same network prefix). This ensures only a limited increase in the size of the forwarding table as the network grows, because the granularity of the aggregation grows. Furthermore, L3 addresses depend on your physical position in the network.

#### **C. Network Size and Broadcast Traffic**
* **L2:** Broadcast traffic grows proportionally with the number of hosts, creating vulnerabilities to "broadcast storms" caused by closed paths (meshes) or interface malfunctions. The network diameter is limited (IEEE 802.1D suggests no more than 7 cascading switches, though fine-tuning parameters or utilizing IEEE 802.1s can allow for a greater number).
* **L3:** Routers intentionally do not forward broadcast traffic. There are no limits to the network diameter, as multiple routing domains can be used if needed.

#### **D. Mobility and Independence**
* **L2:** Provides seamless mobility at L3. It does not matter where you are plugged in; everything works, and it functions with any network protocol.
* **L3:** Offers no transparency and no mobility.

#### **E. Network Paths and Failure Recovery (Transient)**
* **L2:** Possesses limited routing capabilities because all traffic is forwarded using a single spanning tree. This results in suboptimal paths and unfair usage of network resources, as links outside the spanning tree are completely unused. Failure recovery is slow (taking about a minute), though it is improved with IEEE 802.1w (Rapid Spanning Tree Protocol). During a transient state, data traffic might be flooded, and filtering database entries undergo artificial ageing.
* **L3:** Uses multiple forwarding trees and supports multipath routing. In the event of a failure, the network outage is limited only to a subset of the network or specific destinations, while others are usually only marginally affected.

#### **F. Security**
* **L2:** Provides no network isolation. It is vulnerable to ARP spoofing attacks and MAC flooding attacks because the filtering database is populated automatically. Access Control Lists (ACLs) must be defined "per MAC". Furthermore, L2 devices lack the intelligence to operate "per TCP/UDP port" or "per application".
* **L3:** Offers network isolation for better security. IP addresses are explicitly configured by the network manager. ACLs can be defined "per network," and there is usually more intelligence available to operate at the transport and application layers. Many parameters (like firewalls and ACLs) are bounded directly to your address.

### **3. Network Design Choices & Performance Context**

* Because "a router is slow" and "a switch is fast," historical solutions to network performance included using L2 wherever possible (limiting L3) or developing faster new technologies like Asynchronous Transfer Mode.
* **Modern Common Choices:** Simplicity and costs are usually the most important parameters.
    * **Access Layer:** Handled at L2 (using VLANs). Very unusual to see L3 here.
    * **Backbone/Core:** Usually still L2, but sometimes L3. L2-L3 Multilayer switches are placed here to speed-up VLAN interconnection.
    * **Edge/Exit Gateway:** L3 is strictly used for the exit gateway, data centers, WAN connections, and VLAN interconnection.
    * **L4-L7 Services:** Deployed at the exit gateway (for protection) and the data-center (for load balancing, QoS, etc.).

### **4. Multilayer (Layer 3) Switches**

**Definition & Architecture:**
* A "Layer 3 switch" is effectively a router. It is a marketing term invented in the late 1990s because "switch" implies faster speeds.
* It integrates both L2 and L3 capabilities within the same physical box. Originally, this was achieved by having one switch card and one router card in the same chassis.
* L3 switches are usually pure hardware devices.
* They integrate routing and switching to eliminate the "one-arm router" bottleneck. Without integration, inter-VLAN traffic across a trunk link must cross the same physical link toward an external router twice.

**Features & Limitations:**
* Highly oriented to corporate needs.
* They typically lack sophisticated routing protocols (like BGP).
* They implement Access Control Lists (ACLs) in hardware for security.
* Additional features can include content accelerators and firewalls.
* They usually have a limited set of network interfaces.

**Configuration Flexibility:**
* Users can configure interfaces in either L2 or L3 mode based on needs.
* Depending on the device, an interface mode might be hardcoded, or it can be dynamically swapped upon configuration (e.g., using the Cisco `switchport` command).
* L2 ports accept configurations like "access/trunk link".
* L3 ports accept configurations like an "IP address".

### **5. Interfaces in an L2-L3 Switch & Traces**

Four distinct types of interfaces can be configured on a multilayer switch.

#### **1. Traditional L2 Interface**
* Pure L2, operating in either access or trunk mode.
* Spanning Tree Protocol (STP) is active.
* It may be associated with an L3 virtual VLAN interface if routing is needed (acting as the internal one-arm router).

```text
! L2 interface
interface fastethernet0
!

! L2 interface connected to a VLAN
interface fastethernet0
switchport access vlan 2
```
**

#### **2. Virtual VLAN Interface (L3)**
* A virtual interface within the switch that connects the "internal router" to the "internal switch" for VLAN interconnection.
* Equivalent to a pure L3 interface (no STP active).
* The VLAN ID is included in the name of the interface.

```text
!
interface vlan 2
ip address 1.1.1.1 255.255.255.0
!
```
**

#### **3. Traditional L3 Interface (Pure L3)**
* A traditional routed interface connected in access mode.
* Represented as an L2 link terminated on an L3 host.
* STP is **not** active. While BPDUs may be received, the interface cannot enter a blocking state. An RSTP switch connects to it through an Edge port.
* Configuration default varies (Cisco devices might default to L3 or L2; use `no switchport` to force L3).

```text
!
interface fastethernet0/0
ip address 1.1.1.1 255.255.255.0
!
interface fastethernet1/0
no switchport
ip address 2.2.2.2 255.255.255.0
!
```
**

#### **4. Sub-interface for VLANs (L3)**
* A pure L3 virtual interface connected in "trunk" mode.
* It terminates a VLAN onto a logical sub-interface.
* The name of the sub-interface (e.g., `.10`) does **not** imply the VLAN ID. The ID must be explicitly configured using the `encapsulation` command.

```text
!
interface fastethernet0/0
no ip address
!
interface fastethernet0/0.10
encapsulation dot1q 10
ip address 10.10.10.1 255.255.255.0
!
interface fastethernet0/0.20
encapsulation dot1q 20
ip address 20.20.20.1 255.255.255.0
!
```
**

*(Note: Not all four interface types may be available simultaneously on a single device, and L2 interfaces cannot accept IP addresses, whereas L3 interfaces must).*

### **6. L4-L7 Multilayer Switches**

* These devices provide Layer 4 through Layer 7 intelligence (e.g., application-layer balancing, firewalls).
* Intelligence is often modularâ€”different intelligence on different linecards (some cards may be purely L2, while others provide L4-L7 features).
* **Benefits:** Requires no separate boxes, provides simple management (one box does all), lowers costs, and increases flexibility.
* Oriented primarily toward the enterprise market.
* **Placement Strategy:** They are ideal candidates near Wide Area Networks (WANs) or core hubs because most internal or internet-bound traffic is concentrated there, making it the optimal location to enforce security policies and balance traffic across servers.

**Final Engineering Rule:** It is critically important to engineer the L3 network while heavily considering the actual L2 topology, particularly the Spanning Tree Protocol (STP) paths.
